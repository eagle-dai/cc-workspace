# 02｜LLM 量化：从交易想法到可验证研究假设

上一讲我们划分了量化研究四类权责边界：Codex 负责代码编写与命令执行、大模型负责文字整理润色、量化验证模块基于固定数据集完成客观校验，最终结论的取舍与判断责任完全归属研究者本人。

在实操过程中很多人都容易走入一个致命的误区：研究者尚未明确「本次究竟要验证什么」，就急于批量运行回测、拟合收益曲线，再依托模型输出反向拼凑逻辑。本讲旨在**建立一套标准化前置流程：先完整、可证伪地写明研究命题，再使用固定样本开展数据校验。**

日常讨论行情时，我们输出的大多是主观观点，例如“均线金叉具备实战价值”。这类表述仅代表主观倾向，经不起严谨追问：验证哪一段资产行情？以何种方案作为对照基准？收益、交易次数达到何种标准才算策略失效？

本讲核心方案为标准化研究假设卡：统一填写验证对象、对比基准、有效 / 失效判定标准、证据不足边界；随后以「3 日均线上穿 7 日均线」为示范规则，在固定价格样本中检验该信号能否长期跑赢简单买入持有策略。

这里需要你提前明确一个关键认知：本讲仅判断这个规则在指定历史区间内是否存在相对超额收益，不代表这个策略未来一定能稳定盈利。

## 1. 交易观点 ≠ 标准化研究假设

交易思路最初往往以一句主观判断呈现，如“均线金叉有用”“价格突破后延续上涨”“模型判断当前趋势偏多”。这类内容具备研究出发点，但不能直接作为可检验的研究假设，因为缺失验证标的、样本区间、对照基准、失效判定四大核心约束。这里我们严格区分“主观想法”与“标准化假设”，避免你在命题模糊的前提下盲目回测。

### 1.1 主观想法与可检验假设的核心差异

量化研究与行情闲聊的本质区别：是否在运行数据前，书面约定完整、客观的检验规则。

- 主观想法：“均线金叉有用”，仅有模糊方向，无任何检验边界，无法证伪。

- 标准化假设：基于数据集 `data/prices.csv`，检验 3/7 日均线交叉策略收益是否优于买入持有；若策略收益低于基准则不支持假设，总交易次数不足 2 次则判定为证据不足。

这里面蕴藏着两个核心底层规则：

- **可证伪性**：假设必须存在被样本数据推翻的可能。仅描述成功条件、不提前定义失效标准的命题，不具备量化检验价值。

- **零假设**（本讲简化落地规则）：默认基准结论为「均线交叉策略无法跑赢买入持有」，只有足量样本回测证明该策略产生稳定正向超额时，才临时推翻默认立场。本讲不引入复杂统计模型，但强制要求在假设卡写明默认立场，杜绝将样本内偶然收益直接定义为“策略永久有效”。

本讲我们统一采用买入持有作为基础对照基准：样本资产期初收盘价 8.69，期末收盘价 13.58，经计算买入持有基准收益为 56.27%。任何均线策略收益低于该数值，都可以判定在本历史区间无超额优势。

掌握基础概念后，我们来梳理四类典型不合格假设表述，直观识别命题漏洞。

## 2. 六要素标准化研究假设卡

明确无效表述特征后，我们这一节来落地完整的可检验命题模板。假设卡并不是形式化的文档，而是数据运行前签署的研究合约：字段缺失则禁止启动回测；无预设否定条件，则数据结果无客观解读依据。

### 2.1 可检验假设的六大必填要素

表 2-2 是逐项可核对的标准化命题框架，六要素缺一不可。其中第六条，否定条件必须拆分“不支持假设”“证据不足” 两类，不能合并为模糊的“优化参数”。

- 不支持假设：策略收益低于对照基准。

- 证据不足：全样本内总交易次数小于 2 次（`trade_count < 2`）； 最大回撤仅作为客观事实记录，若需将回撤纳入判定标准，必须在假设卡预先写明阈值。

<div align="center">

| 要素 | 要回答什么 | 坏写法 | 好写法 |
| --- | --- | --- | --- |
| 对象 | 哪种资产、哪份样本 | 看 BTC 会不会涨 | `WEB3-DEMO/USDT` + `data/prices.csv` |
| 触发条件 | 什么事件后观察 | 均线变好 | 3 日均线上穿 7 日均线 |
| 观察窗口 | 观察多久 | 后面一段 | 至下一次下穿或样本结束 |
| 对照基准 | 和谁比 | 收益高不高 | 买入持有 |
| 评价指标 | 事先看哪些数 | 效果不错 | 策略收益（`strategy_return_pct`）、买入持有收益（`buy_hold_return_pct`）、策略最大回撤（`maximum_drawdown_pct`）、交易次数（`trade_count`） |
| 否定条件 | 何时不支持、何时说不清 | 尽量优化 | 1️⃣ 不支持：不优于基准 2️⃣ 证据不足：交易数小于 2（`trade_count < 2`） |

**表 2-2　可验证研究假设的六要素**

</div>

六大要素是第 1 讲「研究结论六大边界字段」的落地实现，两套框架一一对应： 资产标的→验证对象；观测区间→观察窗口；信号来源→触发条件；对比基准→对照基准；观测指标→评价指标；推翻条件→否定判定条件。

本讲额外将完整样本文件路径写入验证对象栏，目的是保障我们所有人可以完整复现同一套数据；单独拆分“证据不足”判定，是为了避免交易样本过少时，误以为策略具备稳定规律。

六要素全部填写完整后，我们就可以生成完整可执行的研究假设卡了。

### 2.3 填写假设阶段的人机权责边界

我们结合第一讲划定的四类责任，来明确工具与人工的硬性约束：

1. Codex 仅可协助假设卡排版、执行回测命令；

2. 大语言模型仅能润色调研记录文字；

3. 工具无权在看到回测输出结果后修改预设否定条件，也不能自主调整指标判定档位。

你在向大模型输入素材时，只传入已填写完成的六要素、标准检验问句，千万不要传入回测结果截图；如果模型主动建议更换均线参数、新增评价指标，按照表 2-6「未定义概念偏差」处理，必须新建独立假设卡，不能修改当前假设与既定结论。

### 3.2 标准化三步检验流程

#### 步骤 1：回测前复核完整命题，规避 HARKing 偏差

HARKing（事后假设偏差）：指先查看回测结果，再反向编造验证规则，人为放大策略收益显著性，是量化研究核心大忌。运行代码前，必须对照表 2-2 逐项核对六要素完整度，禁止提前运行会泄露策略收益表现的调试命令；只允许提前读取样本数据、手动核算买入持有基准收益。

#### 步骤 2：执行主假设，预设回测命令

否定条件一经确定不可修改，执行下面的脚本，所有输出只作为客观事实记录，禁止反向修改假设卡：

```bash
python report_cli.py --format summary
```

本讲回测统一简化假设：零交易手续费、零滑点，最终结论必须标注「模型未计入实际交易成本」。输出数据固定读取顺序：

1. 买入持有基准收益 `buy_hold_return_pct`（标准值 56.27%）；

2. 策略收益，对比基准判断超额水平；

3. 策略最大回撤，仅做客观记录；

4. 总交易次数，匹配证据不足判定阈值。 脚本输出的全部警告文本，必须完整保留在事实记录中，不可删减。

#### 步骤 3：完整保存原始回测输出

只基于当前主假设参数保存原始数据，完整复制指标数值与系统警告，禁止中途更换参数、新增评价指标、撰写实盘交易建议。你可以参考下面我给出的基础记录格式：

```text
【主假设行】strategy_return_pct=…%  buy_hold_return_pct=…%  maximum_drawdown_pct=…%  trade_count=…
【事实】（粘贴警告与四指标）
```

补充观测规则（可选）：可同步测试 5/10 均线作为辅助观察数据，但不得更改主假设判断；如果需要将 5/10 均线作为正式检验结论，必须新建独立假设卡。 辅助观测执行脚本：

```bash
python report_cli.py --format summary --short 5 --long 10
```

核心约束：最终结论只采信主假设预设参数输出；辅助观测数据不能将“不支持假设”修正为“策略具备观测价值”。

## 4. 回测结果标准化判读规则

获取数值后，最容易出现的研究偏差并不是计算错误，而是人为将数据解读为假设卡未定义的结论。本节统一规范三段式记录模板，明确事实段、解释段、行动段的书写边界，列明禁止使用的主观表述。

这可以帮助你区分“数据本身”和“你对数据的解读”，防止事后用主观理解给策略贴金，确保每一步都有据可依。

### 4.1 三类标准化判定档位

所有结果统一采用三段式记录：事实段粘贴原始数据与警告；解释段严格匹配表 2-4 三档结论；行动段仅填写继续观测、新建假设卡、终止测试三类指令。

<div align="center">

| 档位 | 何时使用 | 本讲示例 |
| --- | --- | --- |

| 不支持 | 触发「不支持」标准 | 策略收益 ≤ 买入持有（本样本 `56.27%`） |
| 证据不足 | 未触发不支持，但交易数小于 2（`trade_count < 2`） | 交易太少，不得写「规律稳定」 |

- 事实：完整粘贴脚本输出数值与警告原文，不做删减；

### 4.2 完整合规记录示范

```text
【可检验版本】三七均线在 data/prices.csv 中是否优于买入持有？

```

### 4.3 解读阶段常见逻辑偷换与硬性约束

全部解读文字必须能够反向对应六要素或指标边界表，出现下述行为直接判定研究流程失效：

1. 判定逻辑优先级代码化规则：

```text
if strategy_return_pct <= buy_hold_return_pct: reject_hypothesis  # 优先
elif trade_count < 2: insufficient_evidence  # 仅当未触发不支持
if reject_when is empty: reject_design
if metrics_fixed_after_seeing_output: invalid_design
```

2. 流程红线

- 假设卡缺失否定条件：命题设计无效，禁止解读

- 看到回测结果后新增、修改评价指标：研究流程作废

这里我补充一条重要的风险提示：本讲只基于单一资产、短期固定样本开展演示，样本内相对收益不能外推至未来行情。所有解读必须标注「回测未建模手续费、滑点等交易成本」。样本外校验、交易成本测算、组合风控将在第 16-22 讲展开。

手动复核仅校验收益计算口径统一，不代表策略具备实盘交易价值。若脚本输出基准收益与手算 56.27% 不符，优先核对数据集、代码分支、执行命令，禁止直接解读结果。本讲四大核心研究偏差及处理方案：HARKing 事后拟合、事后新增评价指标、缺失对照基准、大模型主观叙事越界的典型表现和处置办法你可以参看表 2-6。

## 总结

这一讲我们走完了完整的标准化研究主线四步流程：依据表 2-2 完善六要素假设卡 → 执行固定回测脚本 → 对照表 2-4 完成三档客观判定 → 复核文档后判定放行 / 退回。

研究全部流程完成后，需完成四项合规校验：第一，全部否定判定规则都在回测前预先定义；第二，核心结论仅依托主假设预设参数的输出数据；第三，结果解读严格限定三类标准化档位，不增加自定义主观评价；第四，假设卡具备完整自洽性，可供他人独立复现全部验证逻辑。
